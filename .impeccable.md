## Design Context

### Users

- Primary audience is mixed equally between web agencies/client reviewers and in-house product teams (PM, design, engineering).
- Product use happens during live-page review and release coordination, where users need to move from capture to decision quickly.
- Core job to be done is one continuous flow: capture visual feedback on live pages, triage pins into action, and collaborate with context until shipped.

### Brand Personality

- Three-word personality: premium, refined, operational.
- Emotional goal is to make users feel in control and confident within the first two minutes of use.
- Tone should stay concise, assured, and purposeful, avoiding playful novelty and empty enterprise jargon.

### Aesthetic Direction

- Direction: premium-refined product UI with warm editorial neutrals and controlled accents.
- Theme strategy: light-first with high-quality dark mode support.
- Motion: expressive where it improves clarity and perceived momentum, always with reduced-motion accommodations.
- Explicit anti-reference: avoid over-minimal layouts that feel empty or under-informative.
- Reference calibration (identity only, not a visual clone): **Linear / Raycast-class** — dense calm surfaces, fast scanning, keyboard-forward patterns where they reduce friction.

### Design Principles

- Build one visible workflow: every major surface should reinforce Capture → Triage → Ship.
- Prioritize confident readability: strong hierarchy, clear grouping, and WCAG 2.2 AA contrast/comprehension standards.
- Keep density intentional: rich information without clutter, avoiding both visual noise and excessive emptiness.
- Use accents as decision signals, not decoration: emphasis should guide status, priority, and next action.
- Craft interactions for operational trust: responsive controls, clear state transitions, and context-preserving details.
- Prefer scannable, low-friction layouts suited to long review sessions and repeated triage.

### Accessibility & inclusion

- Baseline: WCAG 2.2 AA where applicable.
- Respect `prefers-reduced-motion`; decorative motion should degrade gracefully.

## Implementation context (repository)

### Stack

Next.js 16 (App Router), React 19, Tailwind CSS v4, shadcn/ui + Radix primitives, Lucide icons, Supabase (auth/data), Zustand for client workspace state where used.

### Typography

Configured in `apps/web/src/app/layout.tsx` and `apps/web/src/app/globals.css` (`@theme`):

- **Display:** Bricolage Grotesque (`--font-display`)
- **UI body:** Figtree (`--font-sans`)
- **Mono:** Geist Mono (`--font-mono`)

### Color system

OKLCH tokens in `@theme`: warm **paper / ink** ladder and **rule** borders; semantic **`mark`** (workflow attention) and **`ok`** (resolved) accents.
